# Product Design Notes

This directory contains internal product and design notes.

These files are useful when you want to understand:

- why a feature was designed a certain way
- how the management surface was intended to evolve
- how staged modules and backend read models were meant to fit together

These files are usually not the right first stop for:

- setup
- operations
- troubleshooting
- user acceptance testing
